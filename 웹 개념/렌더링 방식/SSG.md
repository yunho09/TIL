# SSG(Static Site Generation)

## 정의

- 개발자가 웹사이트를 배포(빌드)하는 시점에 모든 페이지를 미리 완벽한 HTML로 만들어 두었다가 사용자가 요청하면 즉시 꺼내 주는 방식.
## 작동 원리
### 1단계: 빌드 시점에 미리 만들기

- 개발자가 코드를 다 쓰고 "배포" 버튼을 누르는 순간(빌드 시점) 서버는 모든 페이지의 데이터를 가져와서 수백, 수천 개의 완성된 HTML 파일들을 미리 생성해 둔다.
### 2단계: 서버에 저장

- 만들어진 HTML 파일들은 서버에 보관된다.
### 3단계: 즉시 배달

- 사용자가 특정 페이지를 요청하면 서버는 이미 만들어져 있는 해당 HTML 파일을 그대로 던진다.
- 서버는 별도의 계산이나 데이터 조회를 할 필요가 없다.
### 4단계: 빠른 화면 표시

- 브라우저는 이미 완성된 HTML을 받았으므로 코드를 읽자마자 즉시 사용자에게 화면을 보여준다.
