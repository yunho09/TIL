# ISR(Incremental Static Regeneration)
## 정의

- **미리 만들어둔 페이지** 를 먼저 보여주고 설정한 주기에 따라 서버가 백그라운드에서 내용을 갱신하는 방식.
## 작동 원리

### **1단계: 바로 보여주기**

- 사용자가 페이지를 요청하면 서버는 예전에 **미리 만들어둔 HTML 파일**을 즉시 꺼낸다.
- 사용자는 서버가 계산할 때까지 기다릴 필요 없이 바로 화면을 본다.
### **2단계: 시간 확인하기**

- 서버는 "이 페이지를 만든 지 얼마나 지났지?" 하고 확인한다.
- 개발자가 설정해둔 **유효 기간**이 지났는지 체크한다.
### **3단계: 몰래 새로 만들기**

- 만약 유효 기간이 지났다면 서버는 사용자 몰래 뒤에서 **새로운 데이터**를 가져와서 HTML을 다시 만든다. 
- 이때 들어온 사용자는 아직 갱신되지 않은 기존 화면을 보게 된다.
### **4단계: 교체하기**

- 새로운 HTML이 완성되면 서버는 옛날 파일을 버리고 **새 파일로 갈아끼운다** 
- 그 다음 접속하는 사람부터는 방금 만든 **최신 화면**을 보게 된다.
