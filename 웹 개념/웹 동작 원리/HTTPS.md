# HTTPS(HyperText Transfer Protocol Secure)

## 정의

- HTTP에 데이터 암호화(Secure) 기능을 추가한 프로토콜.
- 브라우저와 서버가 데이터를 주고받을 때 **SSL/TLS 프로토콜**을 사용하여 데이터를 **암호화**한다

## HTTP vs HTTPS 차이점

- 가장 중요한 차이는 **보안(암호화)** 과 **포트 번호** 입니다

| **비교 항목**    | **HTTP (HyperText Transfer Protocol)** | **HTTPS (HTTP Secure)**     |
| ------------ | -------------------------------------- | --------------------------- |
| **데이터 형태**   | **평문 (Plain Text)** 전송                 | **암호화 (Encrypted)** 전송      |
| **보안성**      | 낮음 (해커가 가로채면 내용이 다 보임)                 | 높음 (가로채도 해독 불가능한 난수임)       |
| **포트 번호**    | **80번**                                | **443번**                    |
| **인증서**      | 필요 없음                                  | **SSL 인증서** 발급 및 설치 필수      |
| **속도**       | 암호화 과정이 없어 아주 미세하게 더 빠름                | 암호화/복호화 과정으로 인한 약간의 부하 발생   |
| **SEO (검색)** | 검색 엔진 노출에 불리함                          | **구글 검색 순위 우대 (SEO 필수 요소)** |
