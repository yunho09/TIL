# 정의

* **쿠키 (Cookie)**
    * 서버가 사용자의 컴퓨터(브라우저)에 저장하는 작은 텍스트 파일입니다.
    * 클라이언트가 서버에 요청을 보낼 때마다 자동으로 함께 전송됩니다.

* **세션 (Session)**
    * 사용자의 정보를 사용자의 컴퓨터가 아닌 **서버 측의 메모리나 데이터베이스**에 저장하는 방식입니다.
    * 사용자에게는 저장된 위치를 가리키는 열쇠인 **'세션 ID'** 만 부여합니다.

* **로컬스토리지 (LocalStorage)**
    * HTML5 표준으로 등장한 브라우저 내부 저장소입니다.
    * 쿠키의 단점(용량 제한, 불필요한 서버 전송)을 보완하기 위해 만들어진 **클라이언트 전용 

# 특징

- 쿠키 (Cookie) 
	- **서버 전송:** 매 HTTP 요청마다 헤더에 포함되어 서버로 날아갑니다. (트래픽 발생)
	- **용량 제한:** 약 4KB로 매우 작습니다. 
	* **만료 기한:** 만료 날짜(Expires)를 설정할 수 있습니다. 
	* **보안:** 텍스트가 그대로 노출되어 탈취되기 쉽습니다. 

- 세션 (Session) * 
	- **보안 우수:** 실제 데이터는 서버 금고에 있고, 사용자에게는 번호표(ID)만 주므로 비교적 안전합니다. 
	* **서버 부하:** 사용자가 많아지면 서버 메모리를 많이 차지하여 성능이 느려질 수 있습니다. 
	* **휘발성:** 보통 브라우저를 끄면 세션 연결이 끊깁니다. 

- 로컬스토리지 (LocalStorage) 
	- **클라이언트 전용:** 데이터가 서버로 전송되지 않아 네트워크 대역폭을 아낄 수 있습니다. 
	* **대용량:** 약 5MB~10MB까지 저장 가능합니다.
	* **영구 보존:** 사용자가 직접 지우거나 코드로 지우지 않는 한, 브라우저를 껐다 켜도 데이터가 살아있습니다. 
	* **객체 저장 가능:** JSON 데이터를 저장하기 용이합니다.
# 사용예시

| 종류         | 핵심 용도         | 실무 예시                                    |
| :--------- | :------------ | :--------------------------------------- |
| **쿠키**     | 일시적 상태, 팝업    | - 팝업 "오늘 하루 보지 않음"<br>- 비회원 장바구니(7일간 유지) |
| **세션**     | 보안 중요 정보      | - 은행/관공서 로그인 정보<br>- 관리자 페이지 접근 권한       |
| **로컬스토리지** | UI 설정, 자동 로그인 | - 다크 모드/라이트 모드 설정<br>- 자동 로그인용 토큰(JWT)   |
