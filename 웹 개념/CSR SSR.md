# SSR(Server Side Rendering)
## 정의

- **SSR**은 웹 애플리케이션의 **렌더링을 서버에서 처리**하는 방식입니다
- 사용자가 웹 페이지를 요청하면 서버가 **완성된 HTML**을 생성하여 브라우저에 전달합니다.
## 동작 과정

1. 사용자가 특정 URL을 요청
2. 서버가 해당 요청을 처리하고 필요한 데이터를 가져옴
3. 서버에서 HTML을 완성하여 응답으로 전달
4. 브라우저는 받은 HTML을 즉시 렌더링 → 화면 표시

## 특징

- **장점**
    - 초기 로딩 속도가 빠름
    - SEO 친화적 (검색엔진이 HTML을 바로 읽을 수 있음)
    
- **단점**
    - 페이지 이동 시마다 서버 요청 필요 → 서버 부하 증가

# CSR(Client Side Rendering)
## 정의

- **CSR**은 웹 애플리케이션의 화면을 **브라우저(클라이언트)**에서 JavaScript로 렌더링하는 방식입니다.
- 서버는 기본적인 HTML 뼈대와 JavaScript 파일만 전달하고 브라우저가 JS를 실행해 데이터를 가져와 화면을 구성합니다.

## 동작 과정

1. 사용자가 웹 페이지 요청
2. 서버가 빈 HTML과 JS 파일 전달
3. 브라우저가 JS 실행 → API 호출로 데이터 가져옴
4. 브라우저가 데이터를 기반으로 화면을 렌더링

## 특징

- **장점**
    - 이후 페이지 전환이 빠르고 부드러움 (SPA에 적합)
    - 서버 부하가 줄어듦 (렌더링을 클라이언트가 담당)
    
- **단점**
    - 초기 로딩 속도가 느림 (JS 실행 후 화면 표시)
    - SEO에 불리함 (검색엔진이 JS 실행을 잘 못할 수 있음)
