# SPA(Single Page Application)
## 정의
- SPA(Single Page Application)란 **단일 페이지 모던 웹 애플리케이션을** 뜻합니다.
- **SPA**는 서버에서 완전한 새로운 페이지를 불러오지 않고 **현재 페이지를 동적으로 다시 작성**하는 방식의 웹 애플리케이션입니다.
- HTML, CSS, JS를 한 번 로드한 뒤 이후에는 필요한 데이터만 서버에서 가져와 화면 일부를 갱신합니다.
## SPA 동작흐름
1 . 초기 로드
- 브라우저가 서버에 최초 요청을 보냄.
- 서버는 기본 HTML(`index.html`)과 함께 JS 번들, CSS 등을 내려줌.
- 이때 HTML은 거의 비어 있고 `<div id="root"></div>` 같은 **렌더링 컨테이너**만 존재하는 경우가 많음.

2 . 자바스크립트 실행
- 브라우저가 JS 번들을 실행 → 프레임워크(React, Vue, Angular 등)가 동작 시작.
- 라우터가 URL을 확인하고, 해당하는 컴포넌트를 DOM에 렌더링.
- 즉, **초기 화면은 JS가 DOM을 만들어서 표시**하는 구조.

3 . 사용자 상호작용
- 사용자가 버튼 클릭, 메뉴 이동, URL 변경 등 이벤트 발생.- 브라우저는 서버에 새로운 HTM을 요청하지 않음.
- 대신 **클라이언트 라우터**가 URL 변화를 감지하고 해당 컴포넌트를 다시 렌더링.
 
4 . 데이터 요청
- 화면에 필요한 데이터는 서버 API 호출(fetch, axios 등)로 가져옴.
- 서버는 JSON 같은 데이터만 응답.
- JS가 받은 데이터를 DOM에 반영해 화면 일부만 갱신.
   
5 . 화면 갱신
- DOM 일부만 교체되므로 전체 페이지 새로고침 없이 빠른 전환 가능.
- 사용자는 여러 페이지를 이동하는 것처럼 느끼지만 실제로는 **같은 HTML 문서 안에서 동작**.
