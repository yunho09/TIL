# 서브넷 마스크
## 정의

- 서브넷 마스크는 IP 주소 체계에서 IP 주소의 네트워크 부분(Network ID)과 호스트 부분(Host ID)을 구분하는 32비트 숫자이다.

## 왜 쓰는가?

- **낭비 방지:**
	
    - **과거의 방식:** 예전에는 IP 주소를 A, B, C 클래스라는 **고정된 크기**의 규격으로만 할당했다.
    
    - **문제점:**
        - 예를 들어 직원이 300명인 회사에 C클래스(254개)는 부족하고 B클래스(65,534개)를 주면 **약 65,000개의 IP가 사용되지 않고 버려지는 낭비**가 발생했다.
        
        - 이로 인해 전 세계적인 **IP 주소 고갈 문제**가 가속화되었다.
	 
- **유연한 할당(CIDR):**
	
    - **현재의 방식:** 낭비를 막기 위해 **CIDR**방식을 도입했다.
    
    - **개념:** 클래스 구분 없이 **필요한 만큼만** IP를 잘라서 할당하는 방식이다.
    
    - **효과:** 직원이 300명이라면 `/23` 대역(510개)을 할당하는 식으로 낭비를 최소화하며 IP를 배분할 수 있게 되었다.

## 표기법
| **구분**  | **10진수 표기법**  | **CIDR 표기법** | **의미**             |
| ------- | ------------- | ------------ | ------------------ |
| **기본형** | 255.255.255.0 | **/24**      | 앞에서부터 24비트가 네트워크다. |
| **확장형** | 255.255.0.0   | **/16**      | 앞에서부터 16비트가 네트워크다. |
| **대형**  | 255.0.0.0     | **/8**       | 앞에서부터 8비트가 네트워크다.  |
