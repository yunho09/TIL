# 개념

-  터널링은 직접 접근하기 어렵거나 막혀 있는 통신을 다른 통신 경로 안에 넣어서 전달하는 기술이다.

# 예시

예를 들어 외부에서 서버의 3306 포트로 직접 접근할 수 없을 때 
SSH 연결을 통해 서버 내부의 3306 포트에 접근할 수 있다.

### 서버의 포트 상태가 다음과 같을때

- 22번 포트: SSH 접속 가능
- 3306번 포트: 외부 직접 접속 불가능
- 3306번 포트: 서버 내부에서는 접근 가능

외부에서는 DB에 바로 접속할 수 없다.

``` 
내 컴퓨터 → 서버 3306 ❌
```

하지만 SSH 터널링을 사용하면 다음처럼 접근할 수 있다.

```
내 컴퓨터 → SSH 22번 → 서버 내부 localhost:3306 → DB
```

## 터널링 종류

### 1. SSH 터널링

- SSH 연결을 이용해서 막힌 포트에 접근하는 방식. 

```
내 컴퓨터
  ↓ SSH 22번
서버 내부
  ↓
DB 3306
```

## 2. VPN 터널링

- 내 컴퓨터가 다른 네트워크 안에 들어간 것처럼 만들어주는 방식

```
내 컴퓨터
  ↓
VPN 터널
  ↓
VPN 서버
  ↓
회사/학교 내부망 또는 인터넷
```


# 차이

- SSH 터널링 = 특정 포트 하나 연결 (예: 서버 내부 DB 3306 접속)
- VPN 터널링 = 네트워크 전체에 들어가는 느낌 (예: 회사 내부망 접속)
