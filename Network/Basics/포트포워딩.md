# 포트포워딩

- 외부에서 들어온 요청을 내부 네트워크에 있는 특정 기기의 특정 포트로 전달하는 설정

## 필요한 이유

내 컴퓨터나 서버는 보통 사설 IP를 사용한다.

```
내부 IP: 192.168.0.10
```

이 IP는 내부 네트워크에서만 접근할 수 있어서 외부에서는 직접 접속할 수 없다.

그래서 외부 요청이 공유기까지 들어오면 공유기가 그 요청을 어떤 내부 기기로 보낼지 정해줘야 한다.  
이때 사용하는 설정이 포트포워딩이다.

## 예시

내 서버가 내부에서 3000번 포트로 실행 중이라고 하자.

```
내부 서버: 192.168.0.10:3000
```

공유기에 다음처럼 설정할 수 있다.

```
외부 포트: 80
내부 IP: 192.168.0.10
내부 포트: 3000
```

그러면 요청 흐름은 다음과 같다.

```
외부 사용자
→ 공인 IP:80
→ 공유기
→ 192.168.0.10:3000
```

즉 외부에서는 `공인 IP:80`으로 접속하지만 실제로는 내부 서버의 `3000번 포트`로 연결된다.

### 방화벽과 관계

- 포트포워딩을 해도 방화벽이 막고 있으면 접속이 안 될 수 있다.

### 주의점

- 포트포워딩은 외부에서 내부 서버로 들어오는 길을 여는 것이므로 보안에 주의해야 한다.

## 한 줄 정리

- 포트포워딩은 공유기 공인 IP로 들어온 외부 요청을 내부 서버의 사설 IP와 포트로 전달하는 설정이다.


## 느낀점
---
예전에 마크 서버를 열 때 들어봤는데 이번에 정확한 개념을 정리하면서 외부 요청을 내부 서버로 전달하는 중요한 네트워크 설정이라는 것을 알게 되었다.
단순히 서버에 접속하게 해주는 기능이라고만 생각했지만 외부에서 내부 네트워크로 들어오는 길을 여는 것이기 때문에 보안적으로 조심해야 한다는 점도 알게 되었다.
