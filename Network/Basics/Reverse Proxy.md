# 개념

- 사용자의 요청을 서버가 직접 받지 않고 Nginx 같은 중간 서버가 먼저 받은 뒤 백엔드 서버로 전달하는 구조이다.

```
사용자 브라우저
      ↓
Nginx
      ↓
백엔드 서버
```

예를 들어 사용자가 다음 주소로 요청하면:

```
http://localhost/api/users
```

Nginx가 이 요청을 백엔드 서버로 전달한다.

```
http://localhost:8080/users
```

즉 사용자는 백엔드 서버의 실제 주소나 포트를 몰라도 된다.

## 사용하는 이유

- 하나의 주소로 프론트엔드와 백엔드를 연결할 수 있다.
- 백엔드 서버 주소와 포트를 직접 노출하지 않을 수 있다.
