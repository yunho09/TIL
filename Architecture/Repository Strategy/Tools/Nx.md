# Nx
## 정의

- 모노레포 개발 환경을 효율적으로 구축하고 관리하기 위한 **확장 가능한 스마트 빌드 시스템 이다.
## 핵심 기능

### 1. 영향을 받은 것만 계산 (Affected Command)

가장 강력한 기능이다. 내가 코드 한 줄을 고쳤을 때 Nx는 프로젝트 전체를 다시 빌드하지 않는다.

- **작동 방식:** 의존성 그래프를 분석해서 **"방금 고친 코드와 연결된 프로젝트"** 만 딱 찾아내서 테스트하고 빌드한다.
	- **효과:** 빌드 시간이 10분 걸릴 게 10초로 줄어든다.

### 2. 연산 캐싱 (Computation Caching)

- **"한 번 했던 작업은 두 번 다시 하지 않는다."**
- 내가 빌드(Build)나 테스트(Test)를 돌렸을 때 코드가 바뀐 게 없다면 **이전에 저장해 둔 결과(Cache)** 를 0.1초 만에 바로 보여준다.
- 내 컴퓨터뿐만 아니라 팀원들과 캐시를 공유할 수도 있다(Remote Caching). (내가 빌드했으면 옆자리 동료는 빌드 안 해도 됨)

### 3. 의존성 그래프 시각화 (Dependency Graph)

- 프로젝트가 커지면 누가 누구를 참조하는지 머리로 기억하기 힘들다.
- `nx graph` 명령어를 치면 현재 프로젝트 구조를 그림으로 보여준다.

## 구조적 특징 (Apps vs Libs 패턴)

Nx 모노레포의 핵심은 **"앱은 껍데기일 뿐 실제 로직은 라이브러리에 있다"** 는 것이다.

- **apps/** (애플리케이션 폴더)
    
    - 실제로 배포되는 실행 가능한 단위 (Next.js, React App 등).
    
    - 여기에 코드를 많이 짜면 안 된다. 단순히 `libs`에 있는 블록들을 가져와서 조립하고 설정하는 역할만 한다.
    
- **libs/** (라이브러리 폴더)
    
    - 실질적인 **비즈니스 로직 UI 컴포넌트, 유틸리티 함수**가 사는 곳.
    
    - Nx는 이 `libs`를 잘게 쪼갤수록 캐싱 효율이 높아져 빌드가 빨라진다.
