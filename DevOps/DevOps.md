## 정의

- **DevOps**는 `Development`(개발)와 `Operations`(운영)를 합친 말
- **개발한 코드를 실제 사용자가 접속할 수 있는 서버 환경에 안정적으로 배포하고 운영하는 과정**

## DevOps가 왜 필요할까?

개발한 코드는 내 컴퓨터에서만 실행되면 실제 서비스가 될 수 없다.
사용자가 접속할 수 있도록 서버에 올리고 실행하고 문제가 생겼을 때 해결해야 한다.
DevOps는 개발한 프로젝트를 실제 서비스처럼 배포하고 운영하기 위해 필요하다.

## DevOps가 하는 일  
  
DevOps는 프로젝트가 서버에서 안정적으로 실행되도록 관리한다.  
  
주요 작업은 서버 설정, 배포, Linux 명령어 사용, 네트워크 설정, Nginx 설정, Docker 사용, CI/CD 자동화, 로그 확인, 오류 해결 등이 있다.  
  
즉, DevOps는 개발한 코드를 서버에 배포하고 안정적으로 운영하는 과정이다.
