# Virtual DOM (가상 돔)

## 정의

- **DOM (Document Object Model)**
    
    - 브라우저가 화면을 그리기 위해 가지고 있는 HTML 요소들의 트리 구조입니다.
    
- **Virtual DOM (가상 돔)**
    
    - 실제 DOM을 추상화하여 메모리에 저장해 둔 **자바스크립트 객체(Object)** 입니다.
    
    - 실제 화면에 렌더링되지 않는, **'DOM의 가벼운 사본'** 입니다.
     

## 작동원리

React가 데이터 변경을 감지하고 화면을 업데이트하는 **3단계 과정**이다.

1. **렌더링 (Render):** 데이터(State)가 바뀌면 새로운 Virtual DOM을 생성합니다.
    
2. **비교 (Diffing):** 이전 Virtual DOM과 새로운 Virtual DOM을 비교하여 **변경된 부분(Diff)** 만 찾아냅니다.
    
3. **패치 (Patch):** 찾아낸 변경 사항만 실제 Real DOM에 한 번에 적용합니다.

## 비교표

| **비교 항목** | **Real DOM (실제 돔)**                 | **Virtual DOM (가상 돔)**         |
| --------- | ----------------------------------- | ------------------------------ |
| **쉬운 비유** | **실제 사는 집** 🏠<br>(구조를 바꾸려면 대공사 필요) | **집 설계도** 📜<br>(연필로 쓱쓱 고치면 됨) |
| **실체**    | 브라우저 엔진의 HTML 태그                    | 메모리 상의 자바스크립트 객체 (JSON)        |
| **수정 비용** | **비쌈** (화면을 새로 그림 / Reflow)         | **저렴** (단순 데이터 연산)             |
| **업데이트**  | 변화가 생길 때마다 즉시 갱신                    | 변화를 모아서(Batch) **한 번에** 갱신     |
| **속도**    | 잦은 업데이트 시 느려짐비                      | 잦은 업데이트에도 빠름                   |
