
## 정의

- 콜 스택(Call Stack)과 태스크 큐(Task Queue)를 관리하여 싱글 스레드 환경에서도 여러 작업을 동시에 처리하는 것처럼 보이게 하는 구조

## 콜 스택, 태스크 큐

- 콜 스택
	- 현재 실행 중인 함수들이 쌓이는 공간
	- 동기 코드(바로 실행 가능한 코드)가 먼저 실행됨
	
- 태스크 큐
	- 비동기 작업의 콜백 함수들이 대기하는 공간
	- 종류:
	    - 마이크로 태스크 큐(Microtask Queue)
	    - 매크로 태스크 큐(Macro Task Queue)
## 동작방식

1. 콜 스택이 비었는지 확인
2. 비어 있으면 **마이크로 태스크 큐**의 모든 작업을 먼저 실행
3. 이후 **매크로 태스크 큐**에서 하나의 작업을 꺼내 실행
4. 이 과정을 계속 반복
