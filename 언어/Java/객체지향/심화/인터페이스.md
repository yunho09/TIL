# 인터페이스

## 정의

- 클래스가 반드시 구현해야 하는 메서드의 규칙(설계도)
- 내용은 없고 기능의 틀만 정의

## 예제

```java
interface Animal {
    void sound();  // 추상 메서드 (내용 없음)
}

class Dog implements Animal {
	@Override
    public void sound() {
        System.out.println("멍멍");
    }
}
```

## 특징

### 1. 모든 메서드는 기본적으로 추상 메서드

```java
void run(); // = public abstract void run();
```

### 2. 변수는 상수만 가능

```java
int MAX = 10; // = public static final int MAX = 10;
```

### 3. 다중 상속 가능

```java 
class Cat implements Animal, Pet {
}
```

- 인터페이스는 다중 상속 가능

## default

### default

```java
default void eat() {
    System.out.println("먹는다");
}
```

- 인터페이스 안에 구현이 있는 함수
- 기존 코드 안 깨지게 기능 추가하려고 사용
#### 특징

- 구현 있음
- 오버라이딩 가능
- 객체 통해 호출 가능
