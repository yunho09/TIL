# Speckit
---
- SDD 개발을 쉽게 하도록 도와주는 도구다.

### SDD란
---
- 요구사항을 먼저 명확하게 정의하고 AI와 개발을 진행하는 방식

SDD 개발을 하는 이유는 ai코딩이 점점 강해질수록 코드의 품질이 들쭉날쭉한다.
그래서 ai에게 명확한 요구사항을 정해줘 훨씬 좋은 결과를 얻기 위해 한다.

## speckit 플로우
---
```
Spec
↓
Plan
↓
Task
↓
Implement
↓
Review/Test
```

보통 이 순서로 간다.

### 1. Spec

- spec에는 뭘 만들지를 정의한다.
- 무슨 목표를 가지고 있고 사용자는 누군지 어떤 기능이 있고 어떤 기술스택을 사용할지 작성한다. 

### 2. Plan

- plan에는 아키텍처를 정한다. 
- 폴더 구조를 작성하기도 한다.

### 3. Task

- task에서는 기능을 작게 분리해서 작성한다.

### 4. Implement 

- 여기서부터 ai가 코드를 작성하기 시작한다.
- ai가 앞에서 작성했던 spec, plan, task를 기반으로 코드를 작성한다.

### 느낀점
---
speckit을 사용해서 간단한 프로젝트를 1시간안에 완성해보았다.
해보면서 앞으로 코드를 짜는 실력도 중요하지만 프로젝트의 요구사항을 정의하는 실력이나 설계능력이나 명세를 작성하는 능력도 많이 향상시켜야겠다고 느꼈다. 
그리고 1시간안에 프로젝트가 딸깍하고 나오는 모습을 보니 앞으로의 미래가 많이 걱정된다.
