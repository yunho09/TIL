# 컨텍스트 엔지니어링

## 정의

- 대규모 언어 모델(LLM)이 더 정확하고 원하는 답을 하도록 설계하고 관리하는 기술

## 핵심 구성 요소

- **역할 (Role):** AI에게 부여된 정체성 (예: 10년 차 인사팀장)
    
- **목표 (Task):** 수행해야 할 구체적인 작업 (예: 연차 잔여일 안내)
    
- **조건 (Constraints):** 출력 방식 및 규칙 제한 (예: 표 형식으로, 전문 용어 제외)
    
- **배경 정보 (Context Data):** AI가 모르는 외부 지식 주입 (예: 사내 규정 DB, 개인 정보)
    
- **예시 (Examples):** 원하는 결과물의 샘플 제공 (Few-shot)

## 프롬프트 엔지니어링과 비교

| **구분**     | 프롬프트 엔지니어링           | 컨텍스트 엔지니어링               |
| ---------- | -------------------- | ------------------------ |
| **작업 환경**  | **제미나이 웹사이트/앱**      | **별도 프로그램/시스템**          |
| **데이터 입력** | 사용자가 **직접** 수동 첨부    | 시스템이 DB에서 **자동** 추출 및 조립 |
| **작업 주체**  | 일반 사용자               | 개발자 및 시스템 설계자            |
| **데이터 규모** | 내 컴퓨터 안의 파일 몇 개 (소량) | 회사 문서 및 데이터 전체 (대량)      |
| **작업 방식**  | 말솜씨로 AI를 잘 설득하는 기술   | 데이터 파이프라인을 설계하는 기술       |
| **핵심 이점**  | 즉각적이고 간편한 사용 가능      | 번거로움 및 오답(환각) 방지         |


---
한 줄 정리

프롬프트 엔지니어링은 AI에게 '말'을 잘 거는 기술이고, 컨텍스트 엔지니어링은 AI가 정답을 맞힐 수밖에 없도록 '참고자료'를 자동으로 떠먹여 주는 시스템 구축 기술이다
