# Storybook

## 정의

- UI 컴포넌트를 메인 애플리케이션과 분리하여 독립적으로 개발, 테스트, 문서화할 수 있는 오픈소스 도구이다

## 핵심 기능
### 1. 컴포넌트 격리

- 복잡한 데이터 의존성이나 라우팅 없이 순수한 UI만을 렌더링하여 개발 속도 향상.
    
- `Input`, `Button` 등 원자 단위부터 `Header`, `Page` 등 복합 컴포넌트까지 개별 단위로 검증 가능.

### 2. 상태 시각화

- **Stories:** 컴포넌트가 가질 수 있는 모든 상태를 `*.stories.tsx` 파일에 정의하여 시각적으로 나열함.
    
- **Edge Case 검증:** 실제 앱에서 재현하기 까다로운 상황(서버 에러, 데이터 없음, 로딩 지연 등)을 강제로 주입하여 UI가 깨지지 않는지 확인 가능.

### 3. 인터랙티브 제어

- **Args:** UI 패널에서 Props 값을 실시간으로 변경하며 렌더링 결과를 즉시 확인.
    
- 코드 수정 없이 디자이너나 기획자가 다양한 옵션을 테스트해볼 수 있음.

### 4. 자동 문서화

- 작성된 코드를 분석하여 컴포넌트의 Props 타입, 설명, 예시 코드가 포함된 문서를 자동으로 생성.
